| 文件 / 路径 | 作用 | 修改频率 |
|------------|------|-----------|
| `.vscode/extensions.json` | 推荐其他开发者安装的 VS Code 扩展列表（如 ESLint、esbuild 问题匹配器） | 极少（默认即可） |
| `src/test/extension.test.ts` (你写法为 `extension.test.ts`) | 单元测试文件，用于在 VS Code 模拟环境中测试插件行为 | 开发测试时偶尔修改 |
| `src/extension.ts` (你写法为 `extension.ts`) | 插件逻辑入口，包含 `activate` 和 `deactivate` 函数，实现所有功能 | 经常修改 |
| `.npmrc` | npm 配置文件，通常用于锁定 Node 引擎版本 (`engine-strict=true`) | 几乎不改 |
| `.vscode-test.mjs` | 插件测试运行器的配置文件（如指定 VS Code 版本、测试文件模式） | 极少修改 |
| `.vscodeignore` | 打包 `.vsix` 时排除的文件/文件夹列表（如 `node_modules`、测试文件） | 偶尔修改（增加大文件排除规则） |
| `CHANGELOG.md` (你写法为 `CHANGELOH.md`) | 版本更新日志，发布到市场时会被展示给用户 | 每次发布前必须修改 |
| `esbuild.js` | 使用 esbuild 打包插件的配置文件（控制入口、输出、压缩等） | 极少修改（除非定制构建流程） |
| `eslint.config.mjs` | ESLint 代码规范配置，定义代码风格和规则 | 偶尔根据团队规范调整 |
| `package.json` | 插件清单：名称、版本、激活事件、贡献点、依赖、脚本等 | 经常修改（添加命令、依赖等） |
| `tsconfig.json` | TypeScript 编译配置（输出目录、编译目标、模块解析等） | 极少修改（默认即可） |

> **说明**：  
> - `src/extension.ts` 是实际路径，根目录下的 `extension.ts` 仅为示意（生成器会放置在 `src/` 中）。  
> - `CHANGELOH.md` 是拼写错误，正确应为 `CHANGELOG.md`，但表格保留你的原始写法以便对照。  
> - 未在列表中但常见的 `.vscode/launch.json`、`.vscode/tasks.json` 由生成器自动创建，用于调试和构建任务，一般无需手动修改。
